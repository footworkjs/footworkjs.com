Who created footworkjs?

footwork.js is an open source project and was originally created by developer Jonathan Newman (http://staticty.pe). It was initially developed mostly over the latter half of 2014, however work on some of the core concepts and patterns used therein date back to as far as early/mid-2013.

A lot of what has gone into footwork was built from experience and features garnered from other well known libraries and frameworks. Concepts and features such as 'outlets' (inspired from emberjs), routing (mostly inspired from Durandal), and components (concept and code integrated from knockoutjs) were combined to create what has become footwork.

footworkjs needs your support!

Please consider contributing to the project if you find it at all interesting...pull requests are very welcome!

Roadmap

Where is footwork going? This is the current (incomplete, non-prioritized) list of wants:

  • Documentation on the build process so people can contribute more easily.
  • Custom binding for viewModels.
  • More unit tests (currently very very few tests).
  • RESTful model support with automatic viewModel binding.
  • Routerless outlets.
  • <component /> runtime composition, see this thread on github.
  • Named routes with defined/named (object based) parameter passing.
  • Modularized feature set so as to avoid naming collisions and allow for custom builds
  • Ready-to-go environments (amd, globals, commonjs)
  • Some method of overall app initialization detection (ie: events triggered depending on what components have loaded).
  • Create sub-router example/tutorial
  • IE8 support/fixes (current build has issues)
  • Please feel free to contribute by making a suggestion at github issues!
