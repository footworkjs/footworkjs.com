Introduction

Getting started with any framework can seem like a daunting task. Often times the initial pain is from the loss in output or other hurdles you experience while your knowledge of (and thus capabilities surrounding) the framework increase. Footwork is built to be easy to work with, but there is always that initial bump where the basics of why/how/etc are assimilated.

Once you get going with Footwork its benefits will really start to shine...the following information and resources will help you get up to speed quickly.

The Basics

Using FootworkJS (or any framework) can be a bit tricky to figure out if you haven't done it before or don't yet know know a frameworks particular nuances. The following sections help to outline the very basics of getting started with FootworkJS.

Downloading

To use FootworkJS you must first download it. This is done in one of the following ways (primarily):

  • Install via Yeoman (recommended) Comes setup with Gulp, RequireJS, LESS, unit testing, and more.
  • Skeleton Application (recommended) (based on the generator) Comes setup with Gulp, RequireJS, LESS, unit testing, and more.
  • Download Manually
  • Download via Bower

Download Manually

You can use the dialog below to download Footwork:

What build do I need?

The default (Build: All, Environment: Development) should work for most people out-of-the-box. It does not require any external dependencies (they are all embedded/built into it).

The options and their meaning:

  • Version: The version of the framework you want, latest version is recommended.
  • Build: This defines what dependencies you want built into Footwork vs which ones you will provide. The easiest is generally either an All or All + History.js build as they do not require any dependencies.
  • Environment: Choose Production if you want a minified build, otherwise choose Development.

Download via Bower

The other (and more preferred) method is to utilize Bower to install/use Footwork.
