An introduction to footwork.js...

footwork.js is a front-end javascript framework based on several open source libraries, most notably including knockout.js, postal.js, lodash, and riveter.

The primary goal of footwork is to provide a well organized, clean and powerful basis upon which web applications can be built, tested, and maintained.

Feature-wise it can be thought of as competing in the same realm as AngularJS, EmberJS, Marionnette, Durandal, and others. It is meant to provide a basis (the footing, if you will) of a web application, based on a combination of several well known open source libraries and design patterns.

Footwork focuses on using pre-existing concepts, libraries, and APIs - while extending and adding in features where appropriate. This helps ease the learning curve as the terminology, concepts, and features are not completely foreign.

With a few (very useful) novel features up its sleeve, a solid design, and consistent API you might just want to check footwork out!

Footwork at a glance...

Features

A broad overview of a few of the key features included with footwork:

  • Custom Tags and Components.
  • Automatic loading views and animations.
  • Declarative Wrapping and 2-Way Bindings.
  • Inter-Module Communication and Dependency Tracking.
  • Automatic Dependency Resolution.
  • Declarative Routing (with unlimited outlet and nested router support).
  • Integrates well with jQuery and others (but doesn't need them).
  • Drop-in replacement for knockout-based applications, add in new functionality organically.
  • ... and a ton more.
