An introduction to footwork.js...

footwork.js is a front-end javascript framework based on several open source libraries, most notably including knockout.js, postal.js, lodash, and riveter.

The primary goal of footwork is to provide a well organized, clean and powerful basis upon which web applications can be built, tested, and maintained.

Feature-wise it can be thought of as competing in the same realm as AngularJS, EmberJS, Marionnette, Durandal, and others. It is meant to provide a basis (the footing, if you will) of a web application, based on a combination of several well known open source libraries and design patterns.

Footwork focuses on using pre-existing concepts, libraries, and APIs - while extending and adding in features where appropriate. This helps ease the learning curve as the terminology, concepts, and features are not completely foreign.

With a few (very useful) novel features up its sleeve, a solid design, and consistent API you might just want to check footwork out!

Features

A broad overview of a few of the key features included with footwork:

  • Custom Tags and Components
  • Declarative Wrapping and 2-Way Bindings
  • Inter-Module Communication and Dependency Tracking
  • Automatic Dependency Resolution
  • Declarative Routing (with unlimited outlet and nested router support)
  • Integrates well with jQuery and others (but doesn't need them).
  • Drop-in replacement for knockout-based applications, add in new functionality organically.
  • ... and a ton more.

Custom Tags and Components

Wrap up your widgets and even entire applications with a component, re-use and re-mix all you want, and nest them to your hearts content:

<body>
  <viewModel module="Header">
    <header>
      <span class="title" data-bind="text: pageTitle"></span>
      <calendar></calendar>
      <user-info></user-info>
    </header>
  </viewModel>

  <main>
    <order-form></order-form>
  </main>
</body>

Declarative Wrapping and 2-Way Bindings

Easily wrap (and automatically instantiate) your markup with logic and behavior:

<viewModel module="Inventory">
  <span>Inventory List</span>
  <ul data-bind="foreach: items">
    <li>
      <span class="sku" data-bind="text: sku"></span>
      <span class="name" data-bind="text: name"></span>
      <button data-bind="click: remove">Remove Item</button>
    </li>
  </ul>
</viewModel>

Inter-Module Communication and Dependency Tracking

  • Broadcastables and receivables (keep your state in sync for you, share data between modules/namespaces).
  • Pub/Sub
  • Commands
  • Events
  • Requests
fw.viewModel({
  namespace: 'Footer',
  initialize: function() {
    // numRemainingTodos will receive its value (and be kept in sync) from the TodoList
    this.numRemainingTodos = fw.observable().receiveFrom('TodoList', 'numRemainingTodos');

    // computed property based off of numRemainingTodos receivable
    this.hasSomethingTodo = fw.computed(function() {
      return this.numRemainingTodos() > 0;
    }, this);

    // Open a namespace channel to the TodoList
    var todoList = fw.namespace('TodoList');

    // send event to TodoList telling it something was 'completed'
    this.finishedSomething = function() {
      todoList.trigger('todoCompleted');
    };
  }
});

fw.viewModel({
  namespace: 'TodoList',
  initialize: function() {
    // numRemainingTodos will be broadcast to (and keep in sync) any receivables listening
    this.numRemainingTodos = fw.observable(10).broadcastAs('numRemainingTodos');

    this.$namespace.event.handler('todoCompleted', function() {
      // Task is done, decrement counter which will then autosyncronize to any receivables listening.
      this.numRemainingTodos( this.numRemainingTodos() - 1 );
    }.bind(this));
  }
});

Automatic Dependency Resolution

Integrated support for RequireJS means footwork can automatically download your dependencies for you when they are needed. Or you can compile it all together for an optimized delivery and user experience.

// Locations registered so footwork knows where to find them
fw.viewModels.registerLocation([
  'SearchForm',
  'ProductInfo'
], '/js/viewModels/');

fw.components.registerLocation('inventory', {
  viewModel: '/components/inventory/Inventory.js',
  template: '/components/inventory/Inventory.html'
});

fw.outlets.registerViewLocation('home-page', '/views/homePage.html');

<body>
  <!-- SearchForm viewModel automatically downloaded and bound -->
  <viewModel module="SearchForm">
    <form data-bind="submit: searchNow"></form>
  </viewModel>

  <!-- inventory component automatically resolved, composed, and inserted in the DOM -->
  <inventory></inventory>
</body>

Declarative Routing (with unlimited outlet and nested router support)

Declarative routers with child support and unlimited number of named outlets they can control.

<body>
  <router module="MainRouter">
    <header>
      <outlet name="pageHeader"></outlet>
    </header>
    <main>
      <div class="main-col">
        <outlet name="bodyContent"></outlet>
      </div>
    </main>
  </router>
</body>

fw.router({
  // ...
  routes: [
    {
      route: '/',
      title: 'Home Page',
      controller: function(routeParams) {
        this.$outlet('pageHeader', 'homeHeader');
        this.$outlet('bodyContent', 'indexPage');
      }
    }, {
      route: '/:warehouse/inventory',
      title: 'Inventory List',
      controller: function(routeParams) {
        if(routeParams.warehouse === 'main') {
          this.$outlet('pageHeader', 'mainHeader');
        } else {
          this.$outlet('pageHeader', 'warehouseHeader');
        }
        this.$outlet('bodyContent', 'inventoryPage');
      }
    }
  ],
  // ...
});
